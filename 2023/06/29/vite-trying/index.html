学习 Pinia 源码时尝试运行Pinia 官网提供的样例，基于 Vite初始化了项目，在初始化 Pinia Store 中遇到的一些问题记录

import alias “@” 的问题

由于 Vite 被设计为框架无关的的开发工具，因此原本属于 Vue CLI 的一些预置配置在 Vite 初始化的 Vue 项目中是没有的，比如引用模块时使用的“@”

这样的用法在一个基于 Vue CLI维护的项目中是很常见的，其中“@”表示项目的“src”目录，但这其实是 Vue CLI 服务内部基于 webpack 的 别名解析 为我们实现的能力
